## 3. The Backend Architecture (Python Microservices)

We structured the backend into distinct operational nodes, which are currently running with **foundational stubs** mapped to our overall implementation plan.

### Core Engine (`core_engine/main.py`)
- Acts as the orchestrating daemon using the `schedule` library.
- Defines an `AICEO` class that runs on an endless loop, triggering:
  - **Content Pipelines:** Every 12 hours.
  - **Dropshipping Market Checks:** Every 6 hours.
  - **Daily Analytics Rollups:** At 23:00 every day.
- Loads configuration via a central `config.yaml`.

### The Brain (`brain/`) (Upgraded to Multi-Agent System)
- **`logger.py`**: A robust JSON-based reporting system (`BrainLogger`) that dumps every system action into time-stamped files inside a `logs/` directory. This is the central source of truth for the dashboard UI.
- **`strategy_adjuster.py`**: The central intelligence core. It has recently been upgraded to a **Multi-Agent Memory Architecture** that generates dynamic system prompts for three specialized AI agents working together:
  - 🛍️ **PRODUCT_AGENT**: Scores product viability (TikTok/Amazon/Trends) and decides whether to SCALE, TEST, or REJECT items.
  - 🎯 **HOOK_AGENT**: Generates and categorizes curiosity-driven or emotional hooks.
  - 🎬 **STRATEGY_AGENT**: Diagnoses bottlenecks (e.g. CTR < 3% vs Watch Time < 50%) and provides actionable A/B testing or strategy pivots.
  - 🧠 **MEMORY COMPONENT**: A continuous learning mechanism that tracks winning/failed hooks and products, injecting past learnings into the prompt so the system never repeats a failure and heavily biases towards proven winning formats.

### Content Factory (`content_factory/`)
- **`ideation.py`**: Implements a `HooksGenerator` that currently operates on templates but is designed to hook directly into the OpenAI API to generate viral Youtube Shorts topics and hooks.
- **`video_generator.py`**: Established the pipeline steps for synthetic media: `synthesize_voice` (for edge-tts/ElevenLabs insertion) and `assemble_video` (for Moviepy/FFmpeg aggregation of visual assets + voice).

### E-commerce Manager (`ecom_manager/`)
- **`product_lister.py`**: Created a `ShopifyManager` class designed to authenticate via REST, push new dropshipping items automatically (`list_product`), and modify parameters like pricing based on AI insights.

### Dashboard Backend (`dashboard/app.py`)
- A **Flask application** that serves the cinematic dashboard template.
- Includes an `/api/stats` JSON endpoint that reads the core engine logs from the `brain/logger.py` outputs, packaging them up along with revenue, views, and order metrics for the frontend real-time tracking display.
